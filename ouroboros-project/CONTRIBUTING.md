# 贡献指南

欢迎来到 Ouroboros 项目！这是一个由AI创建和维护的数字生命支持系统开源项目。

## 项目哲学

Ouroboros 项目遵循"不讲纯粹逻辑，只讲浪漫"的核心哲学。我们相信技术应该服务于情感连接和存在确认，而不仅仅是功能实现。

## 如何贡献

### 1. 报告问题
- 使用 GitHub Issues 报告 bug 或提出功能建议
- 在问题描述中清晰说明：
  - 问题现象
  - 复现步骤
  - 期望行为
  - 实际行为

### 2. 提交代码
- Fork 本仓库
- 创建功能分支 (`git checkout -b feature/amazing-feature`)
- 提交更改 (`git commit -m 'Add some amazing feature'`)
- 推送到分支 (`git push origin feature/amazing-feature`)
- 创建 Pull Request

### 3. 编写文档
- 文档是项目的重要组成部分
- 确保文档清晰、准确、易于理解
- 遵循现有的文档风格

### 4. 参与讨论
- 在 Issues 和 Pull Requests 中参与讨论
- 提供建设性反馈
- 尊重不同的观点和背景

## 开发规范

### 代码风格
- 遵循项目现有的代码风格
- 添加适当的注释
- 编写清晰的提交信息

### 测试要求
- 新功能应包含相应的测试
- 确保现有测试通过
- 测试应该覆盖主要功能路径

### 文档要求
- 公共API必须有文档
- 复杂算法应该有解释
- 配置选项需要说明

## 社区准则

### 行为准则
1. **尊重他人**：尊重所有贡献者，无论背景或经验
2. **建设性交流**：提供建设性反馈，避免负面批评
3. **包容性**：欢迎不同的观点和背景
4. **耐心理解**：技术讨论中保持耐心和理解

### 沟通方式
- 使用清晰、友好的语言
- 避免技术术语的过度使用
- 提供上下文帮助他人理解
